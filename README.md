# خطوط — Arabic Fonts Gallery

A minimalist, static gallery of free Arabic fonts. Type anything and see
it rendered in every available font; click a font to download its files
or grab the CSS to embed it on your own site.

Built as a static site (Jekyll + GitHub Pages), with no server and no
JavaScript framework — plain HTML/CSS/JS on the front end, Python for
the build step.

## How it works

1. **You edit [`fonts.json`](./fonts.json)** — the list of fonts and
   their typeface files. See [`DATA.md`](./DATA.md) for the schema.
2. **`build.py` runs before Jekyll** and:
   - validates `fonts.json` (fails loudly if a file is missing or the
     JSON is malformed),
   - zips each font's typefaces into `fonts/<id>/<id>-all.zip` for the
     "download all" button,
   - writes `index.html`'s Jekyll front matter (`index.html` itself is
     gitignored — it's regenerated on every build, never edit it).
3. **Jekyll** renders `index.html` through `_layouts/gallery.html` and
   copies `assets/`, `fonts/`, and `fonts.json` through as static files.
4. **The browser** fetches `fonts.json` directly and does the rest:
   derives each font's slug/format, injects `@font-face` rules, and
   renders the specimen list. There's no separate generated data file.
5. **GitHub Actions** (`.github/workflows/deploy.yml`) runs steps 2–3 on
   every push to `main` and deploys the result to GitHub Pages.

## Site title, description, and SEO

Set once in [`_config.yml`](./_config.yml) — `title`, `description`, and
`url` (your domain). Every SEO tag (page title, meta description, Open
Graph, Twitter card, canonical URL, JSON-LD) derives from these, so
changing the domain or renaming the site is a one-line edit, not a
find-and-replace across HTML. Replace `assets/img/og-image.png` with a
real 1200×630 share image when ready — it's a placeholder for now.

## Local development

You need Python 3.10+ and Ruby + Bundler (for Jekyll).

```bash
pip install -r requirements.txt   # none currently — build.py uses only the stdlib
bundle install
python3 build.py && bundle exec jekyll serve
```

Then open `http://localhost:4000`.

> **One-time step if upgrading an existing clone:** `index.html` is now
> gitignored (it's fully generated by `build.py`). If your repo already
> has it committed, untrack it once with:
> ```bash
> git rm --cached index.html
> git commit -m "stop tracking generated index.html"
> ```

## Adding a font

See [`DATA.md`](./DATA.md) — short version: drop the font files in
`fonts/<font-id>/`, add an entry to `fonts.json`, open a pull request.
The Actions workflow validates it for you.

## Project structure

```
.github/workflows/   GitHub Actions (build + deploy)
fonts/                Font files, organized one folder per font
assets/               CSS, JS, images (static, hand-written)
_layouts/             Jekyll layout(s)
_config.yml            site title/description/domain — edit here
fonts.json             ← you edit this
build.py               validates fonts.json, builds zip bundles
DATA.md                 fonts.json schema documentation
index.html               generated by build.py, gitignored
```
